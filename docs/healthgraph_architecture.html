A question, end to end

Following one message through the system

The message arrives

Someone types a question in LibreChat. The transcript — this turn plus everything before it — is posted to the OpenAI-compatible bridge.

Memory resolves the follow-up

The question gates through memory first. A fresh question passes straight through; a follow-up like which of those leave a balance? is rewritten into a standalone question using the entities in focus.

The router picks a path

Counting and aggregate questions go to text-to-Cypher for exact numbers. Behavioural and “which patients like…” questions go to the hybrid retriever.

The graph is read

The query runs against Neo4j. Reference nodes return friendly names, so a cohort comes back as BCBS Affiliated, Inc. rather than bai. Nothing is answered from the model’s own memory.

The answer is grounded

The model writes the answer strictly from the rows returned, with the Cypher and row count attached as evidence — every figure traces to a query you can see.

Memory updates for next time

The entities the answer was about become the new focus. The next follow-up inherits that context, and the loop continues.

Two jobs, one graph

Interrogation, build, and attestation

The same grounding discipline serves three distinct flows.

Runtime · loop

Interrogation

One question, one grounded answer, context carried forward. The conversational core that lets you validate numbers and ask why.

Build · one-time

Preparation

Ingest, vectorize, cluster, and catalog. Turns raw data into a self-describing, semantically-indexed graph before it serves anything.

On demand · sweep

Attestation

A full pass over the figures, narrated into a validation report. Not part of the conversation — invoked deliberately when you need the whole picture.
